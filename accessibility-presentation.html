From detection to resolution

The Tool Doesn't Just Identify Problems — It Helps Resolve Them

Low Colour Contrast
WCAG 1.4.3 — Contrast (Minimum)
Critical
Form label text uses #9CA3AF on a white background, achieving a contrast ratio of 2.8:1. The minimum required ratio is 4.5:1. Users with low vision or in bright sunlight cannot read these labels.
Fix →Change label colour to #6B7280 or darker to achieve at least 4.5:1 contrast ratio. Test with a contrast checker before deploying.
Missing Alt Text on Images
WCAG 1.1.1 — Non-text Content
High
The document upload icon and the service status illustration have no alt text attribute. Screen reader users will hear "image" with no context, making it impossible to understand the interface element's purpose.
Fix →Add descriptive alt text: alt="Upload document" for the icon. For decorative images, use alt="" to instruct screen readers to skip them.
Keyboard Navigation Blocked
WCAG 2.1.1 — Keyboard
High
The "Submit Application" button and custom dropdown menus are not reachable using the Tab key. Users who cannot use a mouse — including many users with motor disabilities — cannot complete the application.
Fix →Ensure all interactive elements are focusable with tabIndex="0". Replace custom dropdowns with accessible <select> elements or add ARIA role="listbox" with keyboard event handlers.
Unlabelled Form Fields
WCAG 1.3.1 — Info and Relationships
Medium
Three input fields use placeholder text only, with no associated <label> element. When a user starts typing, the placeholder disappears and the field has no programmatic label — screen readers cannot identify it.
Fix →Add a visible <label for="fieldId"> element above each input. Never rely solely on placeholder text as a label — it disappears on focus and has poor screen reader support.
